## 🎯 개요

**약제 검증 플랫폼**은 건강보험심사평가원 약가마스터를 기준으로 약제목록을 검증하고, 시장 경쟁 구조를 분석하는 통합 플랫폼입니다.

### 핵심 가치
- ✅ **정확한 검증**: 최신 약가마스터 + 과거 이력 기반 정밀 검증
- 📊 **시장 분석**: 동일주성분별 업체수 및 시장집중도 자동 분석
- 🚀 **실무 최적화**: 색상 구분, 우선순위 자동 판정으로 검토 효율 극대화
- 🔄 **자동 업데이트**: 월간 약가마스터 자동 반영

### 적용 대상
- 병원 약제부
- 약품 구매 담당자
- 약가 협상 담당자
- 제약사 시장 분석팀
- 의료정책 연구자

---

## ✨ 주요 기능

### 1️⃣ 코드 검증
- **최신 유효성 검증**: 현재 급여 등재 여부 확인
- **과거 이력 추적**: 삭제/변경된 코드 추적
- **추정 현행코드 분석**: 코드 변경 시 현행 코드 추정
- **누락 탐지**: 동일 성분군 내 누락 코드 자동 검출

### 2️⃣ 시장 분석 (v2.0 신규)
- **동일주성분별 업체수**: 앞 4자리 기준 성분군별 업체수 집계
- **시장집중도 판정**: 독점/과점/경쟁/치열 자동 분류
- **경쟁 환경 파악**: 신규 진입/가격 협상 전략 수립 지원
- **독점 품목 식별**: 대체 가능성 낮은 품목 우선 관리

### 3️⃣ 결과 리포트
- **엑셀 기반 결과**: 6개 시트로 구조화된 검증 결과
- **색상 구분**: 판정 유형별 자동 색상 적용
- **우선순위 표시**: 중요도 기준 정렬 및 강조
- **별도 추출 기능**: 미확인 오류, 과거이력, 독점품목 등 선택적 추출

### 필수 요구사항

#### 1. DB 파일 준비
- **파일명**: `약가마스터_성분중심_보완.sqlite`
- **크기**: 약 55 MB
- **포함 데이터**: 2009~2026년 약가마스터 스냅샷
- **다운로드**: [다운로드 링크 제공 필요]

#### 2. 환경 설정
- **AI Drive 경로**: `/drugapp_outputs/`
- **Python 환경**: Python 3.8+ (AI 에이전트용)
- **브라우저**: Chrome, Edge, Firefox 최신 버전 (웹 플랫폼용)

### 설치 방법

#### 방법 1: 웹 플랫폼 사용
1. [https://jooyounkim81.github.io/checkdrugilsan/](https://jooyounkim81.github.io/checkdrugilsan/) 접속
2. "DB 파일 등록" 메뉴에서 SQLite 파일 업로드
3. "파일 검증" 메뉴에서 약제목록 엑셀/CSV 업로드
4. 검증 실행 후 결과 다운로드

#### 방법 2: AI 에이전트 사용
1. AI Drive에 DB 파일 업로드 (`/drugapp_outputs/약가마스터_성분중심_보완.sqlite`)
2. 약제 검증 에이전트 실행
3. 약제목록 파일 업로드
4. 검증 결과 자동 생성

## 📖 사용 방법

### 입력 파일 준비

**지원 형식**: Excel (.xlsx, .xls), CSV

**필수 컬럼** (헤더명 자동 인식):
- 주성분코드 (9자리)
- 주성분명
- 투여경로
- 제품명 (선택)
- 업체명 (선택)
- 규격 (선택)

**예시**:
```
주성분코드,주성분명,투여경로,제품명,업체명
141101ATB,Acetaminophen,경구,타이레놀정,한국얀센
```

### 검증 실행

#### 웹 플랫폼
```
1. DB 파일 등록 → SQLite 파일 업로드
2. 파일 검증 → 약제목록 파일 선택
3. 옵션 선택:
   ☑ 누락리스트 자동 생성
   ☑ 과거이력 추정 현행코드 분석
   ☑ 앞 4자리 동일주성분 확인
   ☑ 동일주성분별 업체수 분석 (신규)
4. 검증 실행
```

#### AI 에이전트
```
"약제목록 파일 검증해주세요"
[파일 업로드]
→ 자동으로 모든 분석 수행
```

## 📊 결과 해석

### 결과 파일 구조

검증 완료 시 아래 6개 시트를 포함한 엑셀 파일이 생성됩니다:

#### 1. 행별검증
각 약제코드의 검증 결과를 행 단위로 표시

**주요 컬럼**:
- `정확성 판정`: 최신 유효 / 참고사항 / 과거 이력 존재 / 미확인 오류
- `동일주성분_업체수`: 해당 성분군의 업체수 (v2.0 신규)
- `시장집중도`: 독점/과점/경쟁/치열 (v2.0 신규)
- `추정 현행코드`: 코드 변경 추정 결과
- `추정 신뢰도`: 높음/중간/낮음

**색상 규칙**:
- 🟢 **무색**: 최신 유효 (조치 불필요)
- 🟡 **연두**: 유효하지만 참고사항 있음
- 🔵 **하늘**: 과거 이력 존재, 추가 검토 필요
- 🟣 **연보라**: 후보 코드 존재, 대조 필요
- 🟡 **노랑**: 미확인 오류, 필수 확인 대상

#### 2. 요약
전체 검증 결과 통계 및 핵심 지표

**포함 내용**:
- 총 입력 건수, 판정별 건수
- 성분군 누락 요약
- **전체 성분군 수, 평균 업체수** (v2.0 신규)
- **독점/과점/경쟁/치열 성분군 수** (v2.0 신규)
- 색상 가이드

#### 3. 동일주성분별_업체수 (v2.0 신규)
주성분코드 앞 4자리 기준 성분군별 업체수 분석

**컬럼 구성**:
| 컬럼 | 설명 |
|------|------|
| 주성분군(앞4자리) | 예: 1411 (Acetaminophen 계열) |
| 대표_주성분명 | 해당 성분군의 대표 성분명 |
| 총_제품수 | 등재된 전체 코드 수 |
| 업체수 | 중복 제거 후 업체 수 |
| 업체목록 | 업체명 리스트 (쉼표 구분) |
| 시장집중도 | 독점/과점/경쟁/치열 |

**색상 규칙**:
- 🔴 **빨강**: 업체수 10개 이상 (치열한 경쟁)
- 🟠 **주황**: 업체수 5~9개 (경쟁 시장)
- 🟢 **연두**: 업체수 2~4개 (과점)
- ⚪ **회색**: 업체수 1개 (독점)

**활용법**:
- **독점 품목**: 대체 불가능, 가격 협상력 낮음
- **과점 품목**: 제한적 선택지, 전략적 관리 필요
- **경쟁 품목**: 가격 협상 유리
- **치열 품목**: 최저가 경쟁, 품질 우선 고려

#### 4. 성분군누락
성분군 단위 누락 요약

**내용**:
- 성분군별 제출율
- 필수/확장/참고 누락 건수
- 최고 중요도 기준 색상 표시

#### 5. 누락상세
코드별 누락 내역 및 중요도

**중요도 분류**:
- **상** (노랑): 필수 누락, 즉시 검토 필요
- **중** (하늘): 확장 누락, 우선 검토 권장
- **하** (회색): 참고 누락, 필요 시 검토

#### 6. 직접변경후보
코드 변경 이력이 확인된 건

**포함 정보**:
- 이전 코드 → 현재 코드
- 변경 시점 (스냅샷 기준)
- 코드 변경 사유

## 💡 실전 활용 사례

### Case 1: 신규 약품 도입 검토
**상황**: 신규 항생제 도입 여부 판단

**활용**:
1. 해당 성분군의 업체수 확인 → 경쟁 강도 파악
2. 누락상세에서 미도입 제품 확인
3. 독점/과점 품목이면 가격 협상력 고려

### Case 2: 가격 협상 전략 수립
**상황**: 연간 약품 구매 계약 갱신

**활용**:
1. "동일주성분별_업체수" 시트에서 치열/경쟁 품목 필터링
2. 업체수 많은 품목 우선 협상
3. 독점 품목은 장기 계약 고려

### Case 3: 코드 오류 일괄 수정
**상황**: 기존 약제목록 검증 후 오류 정정

**활용**:
1. 행별검증에서 "과거 이력 존재" 건 확인
2. 추정 현행코드 및 신뢰도 확인
3. 신뢰도 "높음" 건은 자동 반영
4. 신뢰도 "낮음" 건은 수동 확인

### Case 4: 시장 분석 리포트 작성
**상황**: 특정 치료 영역 시장 구조 분석

**활용**:
1. 해당 성분군만 필터링
2. 업체수 추이 확인
3. 신규 진입/퇴출 업체 파악
4. 시장 집중도 변화 분석
